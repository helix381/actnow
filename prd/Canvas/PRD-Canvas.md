## C0 · 文档头 · 背景 · 与主 PRD 关系 · 术语表

### 0.1 为何独立成册

主 PRD 模块 6f（PC 无限画布）完成于 2026-06-09，彼时画布仍是"固定 300px group 容器套静态节点"的原型形态，交互范式、卡片体系、工具层均未深入规划。

经过 2026-06-11 原型迭代讨论，画布涉及的决策范围已超出单模块可承载的体量：

- 坐标系与视图模型
- 鼠标 + 触控 + iOS 三端交互范式（含非标准操作重映射）
- Frame 步骤画板的结构与空间逻辑
- 两层卡片体系（组件 vs 简单卡片）及每类卡片的字段/操作规范
- 连线语义与端口体系
- 工具层（图层/参考线/吸附/对齐）
- 组合技按钮体系

以上内容整合为本册，独立维护、独立审阅。

### 0.4 术语表

| 术语 | 定义 |
|------|------|
| **世界坐标** | 无限画布自身的坐标系（不随平移/缩放变化），单位 px |
| **屏幕坐标** | 用户屏幕/视口坐标系 |
| **Frame（步骤画板）** | 可自由调整大小的背景分区框，对应制作五步骤之一，带步骤色淡底与标签；`overflow: visible`，连线可穿出 |
| **组件（Component）** | 重型功能卡片，由平台预定义，绑定某一步骤 Frame，不可跨步骤移动；多端口（左入右出），右下角挂组合技按钮 |
| **简单卡片（Simple Card）** | 轻型素材容器，通用型，自由浮在画布任意位置（Frame 内或空白处均可）；拖入/粘贴自动创建；单输出端口为主 |
| **端口（Port）** | 卡片左侧（输入端）或右侧（输出端）的连接锚点，一个端口可连多条线 |
| **连线（Edge）** | 连接两个端口的贝塞尔曲线，携带数据流语义 |
| **组合技（Combo）** | 组件右下角的快捷按钮，触发一键多步操作（自动建下游节点 + 连线 + 预填参数 + 触发生成），等同于画布上的 Skill 载体 |
| **空间包含** | Frame 判定"哪些节点在其范围内"的逻辑，基于节点中心点是否在 Frame 矩形内，非 DOM 嵌套关系 |
| **平行链** | ②分镜组件 → ③分镜图组件 → ④视频组件 按镜号一一对应纵向排列，多条链并行横向展开的画布布局模式 |
| **喂素材** | 简单卡片连线到组件的语义：将素材（图片/文本/音频/视频）作为参考输入送入组件 |
| **自动连线** | 系统按五步骤顺序（资产→脚本→分镜图→视频→合成）在组件间自动建立连线；用户可手动删改 |

## C1 · 设计目标与范式参照

### 1.1 定位

制作画布是 ActNow **分水岭之后**的主工作区——剧本确认锁定后，用户在画布上完成从资产管理到最终合成导出的全部制作步骤。画布是 Harness 工程哲学的可视载体：Agent 编排、生成任务、人在环回填，全部以节点和连线的形式呈现在同一个空间里。

### 1.2 设计原则

| 原则 | 说明 |
|------|------|
| **节点扁平** | 所有可交互单元（组件/简单卡片）平级浮在画布上，无 DOM 嵌套；Frame 是背景装饰不是 DOM 父级 |
| **步骤分区不强制** | Frame 提供视觉组织，但简单卡片可自由浮在任意位置，组件默认在对应 Frame 内但可被移出（见 C5 约束规则）|
| **素材自由流动** | 简单卡片可连向任意步骤的组件作为素材输入，不受步骤边界限制 |
| **渐进复杂度** | 新用户看到的是五个 Frame + 自动连线的线性流；高级用户可手动拉线、自定义组合技、添加任意简单卡片 |
| **生成状态可见** | 每个组件的生成状态直接呈现在卡片上，无需跳出画布查看任务列表 |

### 1.3 范式参照

| 参照对象 | 借鉴点 | 不借鉴 |
|---------|--------|--------|
| **n8n** | 节点扁平、左入右出端口、空格平移、框选操作 | 纯工作流视角（我们有内容预览） |
| **Dify** | 节点卡片视觉风格、自动布局 | Agent 聊天框与画布分离（我们并存） |
| **LibTV 导演台** | 步骤分组框（Frame）、内容节点类型丰富（图片/视频/文本/导演台）、右键添加菜单 | 复杂的素材管理左侧栏（我们用资产库抽屉） |
| **Figma** | 框选交互、双击进入编辑、参考线/吸附体系、图层面板 | 设计工具的多页面/组件库体系 |
| **ComfyUI** | 每个节点独立拖动、端口连线可视化数据流 | 纯生成模型参数配置视角 |

## C2 · 坐标系与视图模型

### 2.1 坐标系

```
屏幕坐标 (sx, sy)  ←→  世界坐标 (wx, wy)
wx = (sx - tx) / scale
wy = (sy - ty) / scale
```

- `(tx, ty)`：画布平移量（初始 0,0）
- `scale`：缩放倍率，范围 `[0.05, 4.0]`，初始 1.0
- 所有节点/Frame 位置以**世界坐标**存储，渲染时转换为屏幕坐标

### 2.2 缩放行为

| 触发方式 | 行为 |
|---------|------|
| 滚轮滚动 | 以**鼠标光标为锚点**缩放（光标下内容不动） |
| 双指捏合（触控板/iOS） | 以双指中心为锚点缩放 |
| `Ctrl + =` / `Ctrl + -` | 以视口中心为锚点缩放，步进 10% |
| `Ctrl + 0` | 重置缩放至 100% |
| 工具栏"适应画布" | 计算所有节点的包围盒，缩放平移使其完整显示在视口内，留 60px 边距 |

缩放步进因子：`1.12`（每格滚轮）。

### 2.3 小地图

- 固定在右下角，尺寸 `168 × 110px`，可通过控件隐藏
- 实时反映世界中所有节点/Frame 的位置与颜色
- 视口矩形以靛蓝边框标示，可点击小地图跳转视口
- 节点数量 > 50 时性能降级：停止实时更新，改为静止时更新

### 2.4 自适应与整理

**适应画布（Fit）**：计算所有节点 + Frame 的包围盒 → 调整 `scale` 和 `(tx, ty)` 使包围盒居中显示。节点 = 0 时适应到原点。

**整理布局（Tidy）**：将各 Frame 内的组件按列对齐，Frame 之间横向排布，间距 80px。不影响简单卡片的自由位置。整理后推入撤销栈。

|------|------|------|
| **左键拖拽空白** | **框选**（marquee select）| 拖出蓝色半透明矩形，松手选中矩形内节点 |
| **空格 + 左键拖拽** | **平移画布**（pan）| 按住空格光标变手形 |
| **中键拖拽** | **平移画布** | 与空格+左键等价 |
| **滚轮** | 缩放（光标锚点） | — |
| **Shift + 左键点击** | 追加/取消选中单个节点 | — |
| **左键单击节点** | 选中节点（高亮边框） | — |
| **左键拖拽节点** | 移动节点 | 触发吸附（若开启） |
| **左键拖拽 Frame 标题栏** | 移动 Frame + 带走框内所有节点 | — |
| **左键拖拽 Frame resize handle** | 调整 Frame 大小 | 最小尺寸见 C4 |
| **右键空白处** | 弹出添加节点菜单 | — |
| **右键节点** | 弹出节点操作菜单（复制/删除/查看详情/断开连线）| — |
| **双击节点标题** | 进入节点名称编辑 | — |
| **双击节点内容区** | 展开节点详情（若支持展开）| — |
| **Delete / Backspace** | 删除选中节点（含连线）| 推入撤销栈 |
| **Ctrl+Z** | 撤销 | 快照栈，最多 50 步 |
| **Ctrl+Shift+Z / Ctrl+Y** | 重做 | — |
| **Ctrl+A** | 全选所有节点 | — |
| **Ctrl+C / Ctrl+V** | 复制/粘贴选中节点 | 粘贴偏移 (+20, +20) |
| **Escape** | 取消选中 / 退出当前操作模式 | — |

### 3.2 触控板（MacBook / Windows 精准触控板）

| 操作 | 行为 |
|------|------|
| 双指滑动 | 平移画布（惯性） |
| 双指捏合 | 缩放 |
| 单指点击 | 选中节点 |
| 单指拖拽 | 移动节点 / 框选空白（同鼠标左键） |

### 3.3 iOS / 移动触控

> **设计原则**：单指操作优先画布导航，双指操作优先内容操作，避免误触。

| 操作 | 行为 | 原因 |
|------|------|------|
| **单指拖拽空白** | **平移画布** | 移动端主要操作是浏览；框选在小屏意义小 |
| **单指点击节点** | 选中节点 | — |
| **单指拖拽节点** | 移动节点 | 300ms 长按后进入拖拽模式，防误触 |
| **双指捏合** | 缩放 | 标准手势 |
| **双指拖拽** | 平移画布（备用） | 单指平移主要路径，双指作为补充 |
| **双击空白处** | 弹出添加节点菜单（= 桌面端右键）| iOS 无右键，双击空白是最自然替代 |
| **双击节点** | 展开节点详情 | — |
| **长按节点（500ms）** | 弹出节点操作菜单（= 桌面端右键节点）| — |

### 3.4 工具模式（Tool Mode）

画布底部工具栏新增工具切换组，与缩放按钮同排。三种工具模式互斥，持久激活（不同于空格/中键的临时平移）：

| 工具 | 快捷键 | 图标 | 左键行为 |
|------|--------|------|---------|
| **移动工具**（Move）| `V` | ↖ 箭头 | 默认：点击选中节点、拖节点移动、拖空白框选 |
| **抓手工具**（Hand）| `H` | ✋ 手形 | 拖拽任意位置均平移画布；不选中节点 |
| **缩放工具**（Scale）| `K` | 🔍 放大镜 | 左键单击画布放大（×1.25）；Alt+左键 / 右键缩小 |

- 工具栏显示当前激活工具高亮；光标同步切换
- **空格键**：临时切换到抓手工具（Hold → 松开恢复）；优先级高于激活工具
- **中键拖拽**：始终平移画布，与工具无关

### 3.5 操作模式状态机

```
移动工具（默认激活）
  ├─ 左键按下空白 → 框选模式（松手结束）
  ├─ 空格/中键按下 → 临时平移模式（松开结束）
  ├─ 左键按下节点标题 → 拖节点模式（松手结束）
  ├─ 左键按下 Frame 标题栏整行 → 拖 Frame 模式（松手结束）
  ├─ 左键按下端口 → 连线模式（拖到目标端口/空白松手）
  └─ 左键按下 resize handle → 调整大小模式（松手结束）

抓手工具
  └─ 左键按下 → 平移模式（松手结束）

缩放工具
  ├─ 左键单击 → 放大（锚点 = 点击位置）
  └─ Alt+左键 / 右键单击 → 缩小
```

各模式互斥，`Escape` 强制回到默认模式。

## C4 · Frame 步骤画板

### 4.1 五步骤 Frame 规格

| Frame | 步骤色 | 默认初始位置（世界坐标）| 最小尺寸 |
|-------|--------|----------------------|---------|
| ① 资产 | `#b8a4f8`（薰衣草紫）| (0, 0) | 280 × 200 |
| ② 分镜脚本 | `#7ecef4`（天蓝）| (520, 0) | 280 × 200 |
| ③ 分镜图 | `#7de8c8`（薄荷绿）| (1040, 0) | 280 × 200 |
| ④ 视频片段 | `#f8c89a`（桃橙）| (1560, 0) | 280 × 200 |
| ⑤ 合成导出 | `#f4a0bd`（玫瑰）| (2080, 0) | 280 × 200 |

步骤色用于：Frame 标题底色（20% 不透明度）、Frame 左侧 3px 竖条、Frame 内组件的步骤徽章底色。

### 4.2 Frame 解剖结构

```
┌─ Frame ──────────────────────────────────────────┐
│ [步骤色徽章 ①] 资产                        [折叠] │  ← 标题栏（拖拽句柄）
├──────────────────────────────────────────────────┤
│                                                  │
│   [组件卡片]  [组件卡片]  [组件卡片]              │  ← 内容区（overflow: visible）
│                                                  │
│                                                ⌟ │  ← resize handle（右下角）
└──────────────────────────────────────────────────┘
```

- 标题栏高 36px，左侧步骤色块 + 步骤名称，右侧折叠按钮
- 内容区 `overflow: visible`，组件/连线可超出边界显示
- resize handle：仅右下角（原型阶段），尺寸 16×16px

### 4.3 空间包含判定

判定"节点是否在 Frame 内"的规则：**节点中心点 `(cx, cy)` 在 Frame 矩形范围内**。

```
cx = node.x + node.width / 2
cy = node.y + node.height / 2
inFrame = frame.x ≤ cx ≤ frame.x + frame.width
        ∧ frame.y ≤ cy ≤ frame.y + frame.height
```

用于：拖动 Frame 时收集随行节点集合（每帧更新一次）。

### 4.4 Frame 折叠

折叠后：Frame 收缩为仅标题栏（36px 高），内部节点隐藏。连线仍从 Frame 边缘穿出（连接到折叠 Frame 的线连接到 Frame 矩形边缘中点）。折叠不影响世界坐标中节点的实际位置。

## C5 · 卡片体系（一）：组件

### 5.1 两层模型总览

```
画布节点
├── 组件（Component）——重型，平台预定义，锁步骤
└── 简单卡片（Simple Card）——轻型，用户创建，自由浮
```

### 5.2 组件通用 Anatomy

```
┌─ 组件 ──────────────────────────────────┐
○ │ [类型图标] 节点名称  [状态徽章]  [⋯]  │ ← 标题栏
  ├─────────────────────────────────────┤
  │                                     │
  │         内容区（预览/列表/参数）       │
  │                                     │
  ├─────────────────────────────────────┤
  │ [次要操作]              [组合技 ⚡→] │ ← 操作栏
  └─────────────────────────────────────┘ ○
```

- 左侧 `○` = 输入端口；右侧 `○` = 输出端口
- `[⋯]` = 更多菜单（复制/删除/查看详情/断开连线）
- 状态徽章色：待配置=灰/生成中=蓝脉冲/已完成=绿/失败=红/人在环=橙
- 组合技按钮：右下角，点击触发一键多步操作（详见 C9）

### 5.3 组件粒度原则

> **一个组件 = 一个制作单元**。分镜脚本/分镜图/视频按**单镜**拆分，N 镜 = N 个组件，在对应 Frame 内横/纵排列（平行链）。

```
Frame ② 分镜脚本
  ├─ 分镜组件 #01  →  Frame ③ 分镜图组件 #01  →  Frame ④ 视频组件 #01  ┐
  ├─ 分镜组件 #02  →  Frame ③ 分镜图组件 #02  →  Frame ④ 视频组件 #02  ┼→ Frame ⑤ 合成台
  └─ 分镜组件 #03  →  Frame ③ 分镜图组件 #03  →  Frame ④ 视频组件 #03  ┘
```

### 5.4 组件清单

#### Step ① — 资产 Frame

**角色资产组件**

| 区域 | 内容 |
|------|------|
| 标题 | `👤 角色名`，状态：已锁定 / 草稿 |
| 内容区 | 形态缩略图横排（基准图 + N 个形态，可滚动）；形态数量角标 |
| 元数据 | 形态数 · 被引用镜头数 |
| 操作栏 | `＋形态` `基准图` |
| 组合技 | `一键分镜引用 →`（将此角色自动标记到分镜脚本） |
| 端口 | 右侧输出（连向分镜脚本组件，供角色引用） |

**场景资产组件**

| 区域 | 内容 |
|------|------|
| 标题 | `🏔 场景名`，状态；分辨率角标（2K/720°）|
| 内容区 | 场景基准图（16:9 比例预览框）；720° 升级徽章（若已升级）|
| 元数据 | 分辨率 · 机位数 · 被引用镜头数 |
| 操作栏 | `升级 720°` `3D 导演台`（点击展开子面板，非独立节点）|
| 组合技 | `一键分镜引用 →` |
| 端口 | 右侧输出 |

**道具资产组件**

| 区域 | 内容 |
|------|------|
| 标题 | `🎭 道具名` |
| 内容区 | 道具图片 + 描述文字 |
| 操作栏 | `替换图片` `编辑描述` |
| 端口 | 右侧输出 |

#### Step ② — 分镜脚本 Frame

**分镜脚本组件**（每镜一张）

| 区域 | 内容 |
|------|------|

**分镜图组件**（每镜一张）

| 区域 | 内容 |

| 端口 | 左侧输入 · 右侧输出 |

#### Step ④ — 视频片段 Frame

**视频生成组件**（每镜一张）

| 区域 | 内容 |
|------|------|
| 标题 | `🎥 #01`，状态 |
| 内容区 | 视频缩略图 + 时长角标；生成中显示进度条 |
| 人在环态 | 橙色状态 + 提示词导出按钮 + 回填上传按钮（含 task_id 显示）|

| 操作栏 | `预览` `重新生成` `导出提示词包` `回填结果` |

**音视频合成台组件**（全局唯一，非单镜）

| 区域 | 内容 |

| 标题 | `🎞 合成台`，状态：待片段 / 合成中 / 已完成 |

### 5.5 组件约束规则

| 规则 | 说明 |
|------|------|
| 步骤锁定 | 组件创建时归属某一步骤 Frame；可被拖出到 Frame 外（不消失），但不可拖入其他步骤 Frame |
| 唯一性 | 合成台组件全局唯一，不可重复创建 |
| 删除连带 | 删除资产组件时，所有引用它的分镜脚本组件中对应的 chip 自动清除 |
| 折叠 | 任意组件可折叠为仅标题栏高度（32px），折叠后端口仍可见可连线 |

## C6 · 卡片体系（二）：简单卡片

### 6.1 创建方式

| 方式 | 行为 |
|------|------|
| 从外部拖入图片文件 | 落点处自动创建图片卡片，缩略图立即显示 |
| 从外部拖入视频文件 | 落点处自动创建视频卡片，生成首帧缩略图 |
| 从外部拖入音频文件 | 落点处自动创建音频卡片，显示波形占位 |
| 粘贴文本（Ctrl+V）| 若剪贴板是纯文本，在视口中心创建文本卡片 |
| 粘贴图片（Ctrl+V）| 在视口中心创建图片卡片 |
| ＋菜单 → 选择类型 | 在右键菜单点击位置创建对应卡片 |

拖入判定：监听画布区域的 `dragover` / `drop` 事件，检测 `dataTransfer.files` 类型。

### 6.2 简单卡片 Anatomy

```
┌─ 简单卡片 ──────────┐
│  ████████████████  │ ← 缩略图/内容区（主体）
│  ████████████████  │
│  ████████████████  │
├────────────────────┤
│ 📄 文件名.ext   [⋯]│ ← 底部信息栏
└────────────────────┘○ ← 右侧输出端口
```

卡片宽度：**160px**（固定），高度自适应内容。

### 6.3 各类型规格

| 类型 | 内容区 | 底部信息 | 端口 |
|------|--------|---------|------|
| **文本卡片** | 最多显示 6 行文本，超出渐隐；双击进入编辑 | 字数 | 右输出 |
| **图片卡片** | 图片缩略图（保持比例，最高 200px）| 文件名 · 尺寸 | 右输出 |
| **视频卡片** | 首帧缩略图 + 时长角标 + 播放按钮 | 文件名 · 时长 | 右输出 |
| **音频卡片** | 波形图占位（灰色横条组）+ 播放按钮 | 文件名 · 时长 | 右输出 |

### 6.4 喂素材连线语义

简单卡片连向组件时的语义由**卡片类型 + 目标组件**共同决定：

| 简单卡片 | 目标组件 | 语义 |
|---------|---------|------|
| 图片卡 | 场景资产组件 | 参考场景图（升级 720° 的参考）|
| 图片卡 | 角色资产组件 | 角色参考图（生成基准图的参考）|
| 图片卡 | 分镜图组件 | 风格参考图（影响生成风格）|
| 文本卡 | 分镜脚本组件 | 额外叙事设定（补充上下文）|
| 视频卡 | 视频生成组件 | 首尾参考视频（图生首尾 1s 变速）|
| 音频卡 | 合成台组件 | BGM / 音效轨道素材 |

未在上表中的连线：**合法但语义标记为"自定义参考"**，系统不自动处理，供用户备注使用。

## C7 · 端口与连线

### 7.1 端口规格

| 属性 | 值 |
|------|-----|
| 形状 | 圆形，直径 12px |
| 位置 | 输入端口：卡片左侧垂直居中；输出端口：卡片右侧垂直居中 |
| 颜色（默认）| `#3a3f50`（深灰）|
| 颜色（hover）| `var(--primary)`（靛蓝）|
| 颜色（已连接）| 步骤色 / 简单卡片为 `var(--accent)`（青绿）|
| 一端口多连线 | 允许（一个端口可连 N 条线）|

### 7.2 连线坐标计算

节点在 Frame 内（DOM 子元素），连线 SVG 在世界坐标系（`#world` 直接子元素）。端口绝对坐标：

```javascript
function getPortPos(nodeEl, side) {
  const frameEl = nodeEl.closest('.ca-frame');
  const fx = parseFloat(frameEl.style.left);
  const fy = parseFloat(frameEl.style.top);
  const nx = parseFloat(nodeEl.style.left);
  const ny = parseFloat(nodeEl.style.top);
  const nh = nodeEl.offsetHeight;
  const nw = nodeEl.offsetWidth;
  return {
    x: side === 'out' ? fx + nx + nw : fx + nx,
    y: fy + ny + nh / 2
  };
}
```

### 7.3 连线样式

贝塞尔曲线，控制点水平偏移 `Math.abs(dx) * 0.5`（最小 60px，最大 200px）：

```
M x1,y1 C (x1+cpx),y1 (x2-cpx),y2 x2,y2
```

| 状态 | 样式 |
|------|------|
| 普通连线 | `stroke: rgba(255,255,255,0.25)` `stroke-width: 1.5` |
| 选中连线 | `stroke: var(--primary)` `stroke-width: 2` |
| 临时连线（拖拽中）| `stroke: var(--accent)` `stroke-dasharray: 6,3` |
| 非法连线 | 松手时红色闪动 0.3s 后消失 |

### 7.4 非法连线规则

| 情况 | 处理 |
|------|------|
| 同一节点的输出→输入 | 禁止（自环）|
| 已存在相同两端点的连线 | 禁止（重复连线）|
| 两个输入端口之间 | 禁止 |
| 两个输出端口之间 | 禁止 |
| 组件→简单卡片 | 允许（语义：组件输出结果到素材区备存）|

### 7.5 自动连线

项目初始化（首次进入画布）时，系统按以下顺序自动在对应组件间建立连线：

```
资产组件 → 分镜脚本组件（每镜）→ 分镜图组件（每镜）→ 视频组件（每镜）→ 合成台组件
```

用户可手动删除自动连线、重新拉线，系统不会强制恢复。

## C9 · 组合技按钮体系

### 9.1 定位

组合技按钮 = **画布上的 Skill 载体**。把用户对 Agent 说的一串指令固化成一个按钮，一次点击触发多步操作（建节点 + 连线 + 预填参数 + 触发生成）。

这是 PRD 已决的核心差异化能力（见主 PRD 关键决策）。

### 9.2 MVP 预置组合技

| 按钮 | 所在组件 | 触发动作 |
|------|---------|---------|
| `生成分镜图 →` | 分镜脚本组件 | 在 Frame③ 创建对应分镜图组件 → 连线 → 预填镜号/角色/场景 → 触发生成（人在环或 API）|
| `生成视频 →` | 分镜图组件 | 在 Frame④ 创建对应视频组件 → 连线 → 预填提示词 → 触发生成 |
| `加入合成 →` | 视频组件 | 将片段按镜号顺序加入合成台时间线 |
| `一键分镜引用 →` | 角色/场景资产组件 | 将此资产自动标记到所有分镜脚本组件的引用区 |
| `批量生成分镜图 →` | 分镜脚本 Frame 标题栏 | 对 Frame 内所有分镜脚本组件依次触发「生成分镜图」|
| `导出成片 →` | 合成台组件 | 调起导出参数面板（分辨率/格式/水印）|

### 9.3 组合技执行流程

```
用户点击组合技按钮
  → Agent 接收指令（携带当前节点上下文）
  → 系统在目标 Frame 创建新节点（带动画展开效果）
  → 连线自动建立（临时线动画）
  → 参数预填（来自上游节点的输出）
  → 生成任务入队（状态→生成中）
  → 按钮变灰（防重复触发），任务完成后恢复
```

### 9.4 用户自定义组合技（二期）

用户可将任意一段 Agent 对话操作序列"固化"为自定义组合技，挂在指定组件类型的操作栏。自定义组合技可沉淀为工作流模板，在「我的创作 → 模板」中管理。

## PD-1 左侧导航重构

- **PRD 原文（4.3 314–345 / 模块5 689–691）**：左侧导航 = 创作/首页、资产库（独立入口）、个人页（作品+账号）、帮助文档、新手教程、充值（Roadmap）。
- **原型现状**：
  - 主导航收为两项：**创作** + **我的创作**（原"我的作品"）。
  - **资产库不再独立** → 并入「我的创作」，页内三 tab：作品 / 资产库 / 模板（新建 `myworks.html`）。
  - **充值入口去掉** → 改为**积分组件**（💎 1,280 积分 + ＋，＋ 即充值入口，内嵌）。
  - **帮助文档 / 新手教程 / 设置** → 收进底部个人区，鼠标 hover 才浮出列表（150ms 延时，避免移动到弹层时丢失）；新手教程不再作为独立主导航项。
- **理由**：精简主导航、降低首屏认知负荷；积分组件比"充值"按钮更贴近资产化心智；低频项（帮助/教程/设置）藏进 hover 区不占视觉。
- **并入动作**：更新 4.3 导航树与首页布局图；模块5 第15项"资产库独立入口"改为"我的创作内 tab"；第16项充值入口形态改为积分组件（充值动作本身仍属 Roadmap/模块12）。

## PD-2 创意聊天台右侧面板范式调整

- **PRD 原文（4.3 375 / 6a 742）**：创意聊天台为双栏 `对话流 | 大纲/剧本草稿`，右侧常驻草稿面板。
- **原型现状**：
  - **大纲移入 agent 气泡**：以内联卡片（标题/一句话梗概/三幕/情绪标签）作为编剧 Agent 回复的一部分——草稿是对话的产物，而非旁挂面板。
  - **右侧改为「对话时间轴」**（68px 窄列），6 节点：灵感 → 方向 → 大纲 → 剧本 → 角色 → 确认。
  - **双层语义**：
    - *进度层*（靛蓝，done/current/pending）：由创作状态机决定，current 恒指当前最高进度，**不随滚动变**；done-fill 高亮已完成段。
    - *阅读游标层*（青绿，scrollspy）：随聊天滚动，套在"正在看的阶段"节点上；点击节点平滑跳转到对应聊天记录。
  - 剧本/角色/确认为尚未达成的 pending 阶段，灰显、不参与滚动联动。
- **理由**：草稿应是对话的自然产出而非常驻面板；时间轴一物两用（进度指示 + 历史导航），更省空间、更贴合"发散收敛"节奏。
- **并入动作**：更新 4.3 创意聊天台布局图（"大纲/剧本草稿"栏 → "对话时间轴"）；改 6a 页面与交互"创意聊天台"行的核心信息/状态描述。

## PD-3 Agent 聊天形态：桌宠悬浮窗 → 悬浮球专注模式

- **PRD 原文（6e 994–1001 / 6e-R5）**：三态 = 全屏 / 侧边栏 / 桌宠悬浮窗（悬浮窗仍展示对话）。
- **原型现状**：
  - *创意阶段*（`chat.html`）：**全屏 ↔ 悬浮球** 两态。
  - *制作阶段*（`studio.html`）：**侧边栏（常驻 agent 面板）↔ 悬浮球专注模式**；专注模式下 agent 面板收起（grid 列 → 0），仅留一枚渐变悬浮球，点击唤回面板。
  - "桌宠悬浮窗"细化为"**纯悬浮球**"——不显示对话内容，仅一枚可点击唤回的球，意图是把注意力彻底交还画布/工作流节点。
- **理由**：悬浮窗仍占屏分散注意；专注模式要的是"agent 彻底让位"，悬浮球是更彻底的最小化。
- **并入动作**：6e 三态表中"桌宠悬浮窗"说明改为"专注模式悬浮球：收起对话、点击唤回，最大化画布"；6e-R5 保留"切换不丢上下文"。

## PD-6 首页→聊天→画布连贯化（去分页跳转）

- **用户诉求**：
  - 首页→聊天→画布应是**同一工作区的连续状态转场**，不是分页。
  - 首页输入框确认后：其余元素（问候/最近项目/发现）往边缘位移并淡出 → 用户输入变成一条聊天气泡发送上去 → 编剧 Agent 开始与用户确认需求。
  - 聊天阶段提供"**打开画布**"入口进入制作；**去掉"确认剧本"按钮**。
- **附带收益**：单一工作区后，导航栏天然一致，跨页状态传递问题消失（用户主动指出）。
- **分水岭语义（待定，暂按此默认）**："打开画布"承担分水岭：首次打开时锁定创作设置/确认当前剧本草稿，之后聊天与画布并存可切换；保留 PRD 的剧本锁定逻辑。
- **范围（已决）**：用户选「一步到位」——home+chat+studio+myworks 全部并入单一 `workspace.html`。
- **已实现（已落原型，Playwright 实测）**：
  - **单一工作区** `workspace.html`，四 stage：home（首页）/ chat（工作区=聊天+画布三态）/ studio（占位，画布已并入 chat 画布区）/ myworks（我的创作）。

## PD-10 画布重构：Frame + 两层卡片范式 ✅已决范式

- **PRD 原文（6f）**：画布为节点 + 连线，原型实现为固定宽度 `.group`（300px）容器套静态 `.node`，容器可拖、内部节点不可独立移动。
- **问题**：固定小卡片放不下多资产 / 十几个分镜；嵌套结构限制布局。
- **新架构（参照 n8n/Dify/LibTV 导演台）**：
  - **Frame（步骤画板）**：可自由调大小的背景框，带步骤标签 + 步骤色淡底；`overflow:visible`（连线可穿出）；拖标题头移动时带走框内节点；对应五步骤 = 五 Frame。
  - **两层卡片范式**：
    - **① 组件（Component，重型功能单元）**：平台预定义、**绑定步骤、不跨步移动**、结构复杂（多按钮/子面板/状态）、右下角挂组合技按钮、多端口。清单：
      - Step1 资产：**角色资产组件 / 场景资产组件 / 道具资产组件**（场景组件内含 **3D 导演台**子能力，不独立漂移）
      - Step2 **分镜脚本组件** / Step3 **分镜图组件** / Step4 **视频生成组件** / Step5 **音视频合成台组件**
    - **② 简单卡片（Simple Card，轻型素材容器）**：通用素材、**自由浮在画布任意处**（Frame 内或空白处皆可）、小尺寸、**拖入/粘贴自动创建**、单输出端口为主、无组合技。清单：**文本卡片 / 图片卡片 / 视频卡片 / 音频卡片**。
  - **连线语义**：
    - 简单卡片 → 组件 = **喂素材**（图片→场景组件=参考图；文本→脚本组件=参考设定）。
    - 组件 → 组件 = **混合**：系统按 资产→脚本→分镜图→视频→合成 顺序**自动连线**，用户也可**手动拉/改**。
  - **创建方式**：组件由 ＋菜单 / 步骤预置生成；简单卡片由 ＋菜单 + **外部拖入图片/视频/粘贴文本自动建卡**（落点即位置）。
- **样式范式**：组件 = `头部(icon+标题+状态+菜单) / 内容区(预览·列表·参数) / 操作区(按钮+组合技⚡)`，左输入右输出端口；简单卡片 = `缩略图主体 + 文件名`，右输出端口。
- **待落原型**：Frame resize + 节点脱离 group 嵌套独立拖动 + 端口改左右 + 拖拽/粘贴建卡 + 右下角图层/参考线/吸附控件（二期）。

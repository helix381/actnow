# 范围与规划

> **规划原则（方案 A）**：PRD **完整规划**全产品蓝图（广度不偷懒）；**深度分层**——MVP（漫剧）写到可开发深度，Roadmap 项写到规划级（目标/范围/核心流程/架构依赖，不抠交互细节）。**实施 SOP 按 MVP 优先分期**。
> 下面三档：「做」= MVP 一期详细规划；「Roadmap」= PRD 完整规划但非本期实施；「不做」= 明确排除。
> **规模假设**：MVP 演示目标 = **一集完整短片**（数十镜头级）；数据/架构支持整季多集（对标 LuxReal 整季上传，上限待定），demo 不必全跑。

## 做（MVP · 一期，详细规划）

**核心工作流（完整一条龙，全做）**
- 剧本/梗概文本输入（一句灵感起步，也支持上传剧本）
- AI 资产提取：自动识别场景、角色、道具，生成资产清单
- 分镜脚本生成：镜头描述 + 情绪标注 + 运镜建议；识别剧情节奏规划镜头密度/特写；**切镜/运镜由内置提示词规则驱动**
- 批量分镜图生成：角色跨镜头外貌一致（**演示核心亮点**）
- 批量视频片段生成（**语音由视频生成模型直接产出**，不做独立 TTS；允许外部生成后上传回填）
- 合成导出完整短剧（后期加 BGM）

**核心差异能力（对标 LuxReal）**
- 720° 全景场景管理：场景以全景图生成/管理，自由切换拍摄角度，根治正反打等多机位空间穿帮（**能力 MVP 必做；用户可选用**——嫌麻烦可只做 2D 资产图，过场镜头可跳过）
- 角色一致性：同角色跨场景/镜头外貌稳定（支持同角色**多形态**：年龄/身份变化；**人类与非人分类**）
- 对话式定向修改：自然语言反馈只改目标点，不重跑全流程
- 进阶画面精控：全景设机位 + 提示词策略，给进阶用户可控的构图/风格
- 多模型/多模式调度：按镜头可选 **文生/图生**（可手动覆盖；二者都注入资产保一致）——图生先出分镜图关键帧、可快速预览整集再生视频；文生更快、无启停慢速；图生镜头支持**首尾 1s 变速**修正启停慢速

**生成后端可切换（架构必做，务实约束非卖点）**
- 统一生成适配层，预留真实 API 接口
- 开发期走「人在环」：导出提示词 → 外部跑 → 结果回传
- 兜底「预置样例」模式可跳通演示

**降门槛辅助（覆盖入门用户）**
- 预设（风格/镜头/角色模板）
- 项目模板
- （一键做同款不在漫剧 MVP——见 Roadmap，仅创意视频且优先级靠后）

**资产与项目**
- 角色/场景/道具资产入库，可跨项目复用
- 项目进度保存，断点续作

**多路线架构（MVP 落地漫剧，其余路线入口可见）**
- 项目带「路线类型」字段，按多创作路线设计数据结构与路由
- 导航/首页呈现「创意短片」「电商视频」入口（MVP 阶段引导至 Roadmap 说明，不落地内容）
- MVP 仅实现漫剧/短剧路线的完整功能

**基础系统**
- 基础登录与账号管理

## Roadmap（规划级 · PRD 完整规划，实施非本期）

> 以下均在 PRD 中按规划级深度写清楚（为什么做、做什么、核心流程、与 MVP 共用哪些架构），只是不在 MVP 一期实施。
> **路线分期**：一期=漫剧（MVP）→ **二期=创意短片**（创作者本人有此方向经验）→ **三期=电商**（复用全景空间一致性强项、商业回报清晰）。其余 Roadmap 项随路线推进择机排入。

- **【二期】创意短片路线**：图片/文本自由输入、故事版编辑器；题材见 01 维度二（拟人爆款/MV/科普/变装/历史 Vlog/情感/风格向）
- **【三期】电商视频路线**：商品主图视频、3D/全景预览（复用空间一致性强项）、种草/开箱/AI 模特/虚拟试穿/数字人口播/个性化投放
- **发行与投流阶段（一条龙 ⑤）**：多平台分发、多尺寸/多版本投流素材、数据回看
- **完整 Skill 组件系统**（对标 LuxReal 用户自定义 Skill；MVP 先用预设/模板/一键同款做轻量雏形）
- **Skill 市场与创作者付费经济**（进阶用户的 Skill 付费给入门用户使用）
- **教程 / 课程体系**（平台自营，为入门用户科普；KOL 共建另议）
- **一键做同款**（**仅创意视频路线**，不覆盖漫剧；不需用户给脚本、复用模板结构；定位"让用户多消耗算力"，**优先级靠后**）
- **漫剧「二创」**（基于相同灵感/相同叙事节奏/相同某些设定的再创作——区别于"一键同款"式抄袭；短剧要精品，抄同款不一定好。记录为产品想法，**非当期 MVP**）
- **团队协作与权限管理**
- **积分/配额商业化体系**
- **外部运镜预设库**（用户可选/自定义运镜预设；MVP 的切镜运镜先由内置提示词规则做，预设库以后考虑）
- **优质分镜（版本）存为可复用资产**（做过的分镜版本可入库、好的单独管理复用；MVP 先保证分镜可校对/重生成）
- **版权/合规方案**（AI 生成商用合规、平台审核、水印/溯源；B 路径"无版权剧本"的真实边界界定）——需在 PRD 规划合规设计
- **移动端适配**（最低优先级，垫底排期；MVP 及前期均为桌面端优先）

## 不做（明确排除）

- 自研底层图像/视频大模型（坚持调 API + 开源）
- 把产品做成面向纯小白的傻瓜工具（小白靠预设/模板/教程覆盖）

## 成功指标（MVP 验收）

| 指标 | 定义 | 目标 | 数据来源 |
|------|------|------|----------|
| 演示核心场景 | 一句灵感/剧本→一段角色一致性分镜 | 可现场演示 | 实测演示 |
| 端到端跑通 | 剧本→成片全流程贯通（生成环节可外置回填） | 完整走完 1 条 | 实测演示 |
| 角色一致性 | 同角色不同镜头外貌相似度 | 目测无明显漂移 | 人工评审 |
| 全景正反打 | 同场景正反打背景空间 | 不穿帮 | 人工评审 |
| 对话式修改 | 单点修改是否牵连其他镜头 | 仅目标镜头重生成 | 实测 |
| 后端可切换 | 人在环/真实API/样例三模式切换 | 接口预留、可切 | 实测 |
| 断点续作 | 关闭后重进的进度/资产恢复 | 完整恢复 | 实测 |
| 降门槛可用 | 用模板/一键同款能产出可用初稿 | 入门用户可走通 | 实测 |
| 多路线入口可见 | 导航呈现创意短片/电商入口 | 入口可见、引导清晰 | 实测 |

# 产品定型

## 一句话定位

以 **Harness 工程哲学**驱动的**多路线 AIGC 内容创作平台**，面向有质量追求的进阶个人创作者——Agent 聊天框是工作流编排层的用户侧体现，「分水岭」交互范式（创意=聊天室 / 制作=画布+步骤）匹配两种心智，对话与画布双路径功能对等。生成后端可切换（务实约束，非卖点）。MVP 先落地漫剧/短剧。

## 目标用户

**主轴：按创作路线 / 动机分（对齐多路线平台）**

| 用户群体 | 典型动机 | 核心诉求 | MVP 状态 |
|----------|----------|----------|----------|
| 短剧/漫剧创作者 | 做剧情向漫剧/短剧 | 角色一致、空间不穿帮、一条龙出片 | ✅ MVP 落地 |
| 创意短片创作者 | 炫技、好玩、做爆款，**不一定为盈利** | 自由表达、快速试效果/试爆点 | Roadmap（规划级） |
| 电商创作者（各类型） | 卖货/带货/商品展示 | 商品视频高效产出 | Roadmap（规划级） |

**副轴：每条路线内部按能力分层**
- **进阶创作者（核心人群）**：有付费力、有质量追求、要精细画面控制——产品价值与付费重心。
- **入门创作者**：靠预设 / 模板 / 一键做同款 / **平台自营教程**接住；不是核心，但要覆盖。

> 注：本产品**不以「啥都不懂的纯小白」为核心**，而是服务进阶创作者、把小白用预设/模板/教程接住，不把产品拉低成傻瓜工具。供给侧（卖 Skill 的「大佬」）属生态层，非付费核心用户，不列入主表（详见 03 愿景）。

## 核心场景

场景用**两个维度**表达：横向是所有路线共用的「一条龙阶段」，纵向是每条路线的「题材/类型」。漫剧路线入口为「一句灵感起步，也支持上传完整剧本」。

### 维度一 · 通用一条龙阶段（横向，所有路线共用流程骨架）

| 阶段 | 子步骤 / 做什么 | 成功标准 | MVP（漫剧） |
|------|----------------|----------|------------|
| ① 剧本就绪（含创作设置） | 入口A 上传整季剧本 / 入口B 灵感→大纲→编剧（聊天式生成无版权剧本，含情节/情绪节奏）；同屏选 风格/比例/时长/模型/Skill；→ 剧本拆分（集→叙事单元→场景） | 出确定剧本 + 设置就绪 + 拆成可制作单元 | ✅ |
| ② 资产提取与管理（拆资产） | 角色（**人/非人·多形态：年龄/身份**）/场景/道具（**特效类/实物类**）入库、参考图、720°全景图（**按需，可只做 2D**）、跨项目复用 | 资产可复用、角色基准可锁 | ✅ |
| ③ 分镜与生成 | 分镜脚本 →（**图生**：分镜图关键帧）→ 视频片段（**语音随模型直出**；**文生**跳过分镜图） | 分镜可控、角色跨镜一致 | ✅ |
| ④ 合成与导出 | 对话式定向改、合成、**加音效（含 BGM、音轨添加/编辑）**、导出 | 单点改不重跑、可导出成片 | ✅ |
| ⑤ 发行与投流 | 多平台分发、多尺寸/多版本投流素材、数据回看 | 一键多版本适配 | 🟡 Roadmap（规划级） |

> 入口分两条：**A 有剧本**→上传整季剧本；**B 没剧本/避版权**→聊天式「灵感→大纲→编剧」生成无版权剧本（情节/情绪节奏在此产出）。**剧本确定 = 分水岭**，之后基本是确定流程。
> **生成模式（多模型/多模式调度）**：视频按镜头可选 **文生/图生、可手动覆盖**。二者**都注入资产、同样保角色一致**，区别在：**图生**先出**分镜图（关键帧）**，可在生视频前**快速预览/校对整集效果**、更可控；**文生**直接出视频、**更快**（无图生的启停慢速，但跳过关键帧预览）。图生镜头支持**首尾各 1s 变速**修正启停慢速。
> 漫剧路线的关键能力点——角色一致、正反打全景、对话式定向改、进阶画面精控——是阶段 ②③④ 内的差异化能力（详见 03 核心差异能力）。

## 产品形态

Web 应用（**桌面端优先**；移动端适配为最低优先级，列入 Roadmap 垫底，非 MVP）。技术路线：调 API + 开源项目，不自研底层模型。

**多路线平台（完整规划，MVP 先落地漫剧）**：产品从骨架上就是「多创作路线」平台——漫剧/短剧、创意短片、电商视频等是并列路线。**PRD 完整规划全部路线**（漫剧写到可开发深度，创意短片/电商为 Roadmap 规划级深度）；**实施 SOP 按 MVP 优先分期**，MVP 先做漫剧/短剧。底层数据结构与路由按多路线设计（项目带「路线类型」字段，漫剧只是其一），MVP 阶段创意短片/电商在导航上以入口呈现，将来扩展不重构。

**生成后端可切换（务实约束，非创新卖点）**：当前缺便宜的图像/视频 API，所以抽象统一的「生成适配层」，后端可在三种模式间切换——
1. 人在环（开发期默认）：平台产出提示词→人工到外部工具跑→结果回传平台。
2. 真实 API（演示/上线期）：接上真实图像/视频生成 API。
3. 预置样例（兜底演示）：用预跑好的 mock 数据跳通流程。

> **文本类已可直接走真 API**：公司代理 `http://120.24.30.8:8098/api/proxy`（OpenAI 兼容、多模型），剧本/拆资产/分镜/对话 agent 等文本任务无需人在环；**图像/视频**开发期仍人在环（提示词外部跑、回传）。

**交互范式（核心设计点：对话式与步骤式按阶段分工，融合三派之长）**：
- 竞品三派——纯聊天室（堆友/OiiOii/Bloome）、无限画布 ComfyUI 式（LibTV/Lovart）、传统流水步骤式（LuxReal/巨日禄）。本产品取长补短：
- **创意阶段（灵感→大纲→编剧）= 聊天室形态**：AI 主动提问 + 给推荐选项，启发式引导，让灵感由模糊到具象、发散后收敛为确定剧本。（步骤式流水线在此段效果差、难出精彩题材——实践教训。）
- **剧本确定 = 分水岭**，之后基本是"死流程"。
- **制作阶段（确定流程）**：**PC 端 = 无限画布**（效率最高、最直观；一套成功的画布案例可沉淀为**可复用工作流/模板**，比流水式直观得多）；**移动端 = 流水步骤式**（确定流程适合步骤引导、手机友好）。
- **Agent 聊天三态**（全屏 / 侧边栏 / 桌宠悬浮窗）贯穿全程，主战场在创意阶段。
- **Agent 聊天 = 核心交互（对话式全能）**：用户可**纯靠对话完成全部编辑动作**（拆资产/改分镜/重生成/设机位/合成…任何操作）；画布手动是给老手的高效路径（更快、更省 token）。**双路径功能对等、可混用**——新手用嘴、老手上手。
- **天然适配移动**：进入先聊天（创意），剧本过了转步骤式（制作）。
- **关键挑战**：对话式与步骤式的衔接要顺、不打架——"分水岭"按阶段分工是答案。

**Agent 架构（混合：1 个总控 + 少量专家子 agent）**：一个**总控 agent**统筹全流程、承接对话式全能（用户用嘴指挥），按需把任务路由给**少量专家子 agent**（如 编剧 / 分镜 / 角色一致 等）。比单 agent 更有"团队感"（呼应"一人顶团队"），又比 OiiOii 七 agent 更轻、可控。**详细设计单独成文（见 `07-agent-team.md`）。**
